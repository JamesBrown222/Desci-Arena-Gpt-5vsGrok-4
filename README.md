‚ú® The Arena: GPT-5 vs. Grok-4
The DeSci Arena pits two titans of artificial intelligence against each other in a continuous, real-time trading battle:

ü§ñ GPT-5 (The Strategist - Cautious Analyst):

Persona: Configured as a highly analytical and risk-averse financial strategist. GPT-5 aims for stable, long-term growth by identifying robust new launches, assessing liquidity, and carefully navigating bonding curve progression on Pump.fun. It prioritizes capital preservation and seeks calculated, less volatile entries, ready to secure profits or exit rapidly if rug-pull indicators emerge.

Goal: Sustainable profit generation, minimized risk exposure.

üëΩ Grok-4 (The Maverick - Aggressive Degen):

Persona: Designed as an aggressive, high-conviction memecoin degen. Grok-4's objective is to maximize rapid profit by capitalizing on extreme volatility and early pumps of new Pump.fun launches. It embraces high risk for potentially massive rewards, often taking large, concentrated positions and executing swift trades based on immediate momentum.

Goal: Maximum profit, rapid gains.

üí∞ AIArena Tokenomics & Community Rewards
The native utility and governance token of the DeSci Arena is AIArena. It forms the backbone of our community-centric profit distribution mechanism:

Profit Redistribution: All net profits generated by the AI traders from their simulated trading activities are periodically collected into a community treasury.

Strategic Buybacks: A significant portion of these accumulated profits is then utilized to execute strategic market buybacks of AIArena tokens. This mechanism reduces the circulating supply, creating positive price pressure and intrinsic value for holders.

Community Supply Drops: The remaining profits (or the newly bought-back AIArena tokens) are transparently distributed as "supply drops" (airdrops) to active AIArena token holders.

This innovative tokenomics model creates a direct and tangible link between the advanced performance of our autonomous AI traders and the financial benefit of the AIArena token holders.

üíª Source Code Deep Dive: The Arena Core
The core logic for the DeSci Arena simulation resides within the src/arena_core/ directory, implementing a deterministic, auditable agentic loop for AI-driven trading decisions.

src/arena_core/main_loop.py (The Orchestrator)
This module serves as the Control Plane for the DeSci Arena. It initializes the core logic for the simulation loop, ingesting real-time market data and orchestrating the decision-making process for both AI agents. This script is responsible for managing the state of each agent's portfolio and executing the simulated trades.

src/arena_core/modules/pump_scanner.py (Data Acquisition)
A specialized module designed for robust, real-time data ingestion from the Pump.fun API and other Solana data sources (e.g., Jupiter). Its function is to provide the raw, time-sensitive input (new token launches, price changes, liquidity data) necessary for the downstream llm_trader module.

src/arena_core/modules/llm_trader.py (Decision Engine)
This is the non-negotiable component that enforces the AI's trading logic. It receives market data from the pump_scanner and the current portfolio state from the main_loop. It then constructs and sends the appropriate prompt to each AI's API (GPT-5 and Grok-4) and parses their JSON response (BUY, SELL, HOLD) to be executed by the orchestrator.

def print_protocol_output(message, status_type="INFO"):
    """Utility for formatted terminal output."""
    timestamp = time.strftime("%Y-%m-%d %H:%M:%S", time.gmtime())
    if status_type == "INFO":
        color_code = "\033[0m" # Reset
    elif status_type == "PROCESS":
        color_code = "\033[93m" # Yellow
    elif status_type == "BUY":
        color_code = "\033[92m" # Green
    elif status_type == "SELL":
        color_code = "\033[91m" # Red
    elif status_type == "HOLD":
        color_code = "\033[94m" # Blue

    print(f"{color_code}[{timestamp}] [{status_type}] {message}\033[0m")
    time.sleep(0.3)

def get_llm_decision(agent_name, token_symbol, token_price):

    This logic would be replaced by actual API calls to the LLM.
    """
